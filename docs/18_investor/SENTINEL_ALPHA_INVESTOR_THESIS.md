# Sentinel Alpha investor thesis (internal, v1)

## One-line thesis

Sentinel Alpha aims to become a default pre-execution trust layer for agentic onchain actions: a paid decision API plus SDK workflow that helps builders route risky execution paths before funds move.

## Why now

- Agentic transaction flows are moving from demos to production-like usage on public chains.
- Teams are shipping faster than their safety and policy tooling matures.
- x402-style paid API patterns make machine-to-machine security checks commercially viable.
- Infrastructure and identity rails (including ERC-8004) enable higher-trust distribution.

## Market context

- The agentic AI + autonomous systems market is growing quickly, but exact market-size figures vary by source and methodology.
- Security concern levels in agentic systems are widely cited as high; any percentage claims require source verification before external use.
- x402/payment-required patterns are early-stage and still forming standards and adoption norms.

## Agent economy security gap

- Many agent workflows can sign and execute onchain actions faster than human review loops.
- Existing controls are often fragmented across prompts, simulation scripts, and post-incident monitoring.
- Builders need machine-readable pre-execution signals they can enforce in policy code, not only dashboards.

## Sentinel Alpha positioning

- Sentinel Alpha is positioned as a pre-execution risk decision layer for autonomous contract execution paths.
- It is not positioned as a replacement for audits, monitoring, or operational controls.
- It is designed to integrate into builder code where decision outputs can gate or route execution.

## Current proof (live now)

- Live website and public product pages.
- x402-gated API path for risk scoring.
- npm SDK live: `@beezshield/sentinel@0.1.0` (install: `npm install @beezshield/sentinel`).
- AgentKit-style example available in-repo.
- ERC-8004 identity is publicly registered.
- Public docs and trust pages are live.

## Roadmap (not live claims)

- Official provider coming next (AgentKit provider track).
- Bytecode-aware scanner enhancements.
- Sandbox simulation workers for richer pre-execution analysis.
- Reputation / security credit layer for longitudinal trust signals.
- Higher-throughput infrastructure targets for future scale.

## Revenue model

- Usage-priced x402 lanes for API decision requests.
- Tiered pricing by latency/priority and workflow criticality.
- Potential future enterprise contracts for custom policy/ops integration.

## Moat

- Distribution moat from builder integrations and workflow embedding.
- Data moat from feedback loops on decision quality over time.
- Trust moat from transparent claims discipline and identity-backed presence (ERC-8004 + public artifacts).

## Strategic ask

- Introductions to high-signal builder teams (agent execution, trading bots, wallet automation).
- Design-partner conversations for roadmap validation and integration depth.
- Capital + strategic support to accelerate roadmap items while preserving claim discipline.
